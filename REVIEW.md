# Общий обзор проекта (Code Review)

Этот документ содержит обзор проблем, найденных в бэкенде и фронтенде проекта, с рекомендациями по их исправлению. Проблемы разделены по степени критичности.

---

## 1. Критические уязвимости и проблемы

Эти проблемы необходимо исправить в первую очередь, так как они несут прямую угрозу безопасности или стабильности приложения.

| Сервис/Часть | Файл | Проблема | Рекомендация по исправлению |
| :--- | :--- | :--- | :--- |
| **Бэкенд (profile-service, vacancy-service)** | `app/config.py`, `app/utils/security.py` | **Ключевая ошибка: Неработающая аутентификация.** Сервисы падали при запуске или при попытке проверить JWT, так как у них была неполная или некорректная конфигурация (`SECRET_KEY`, `ALLOWED_ORIGINS` и т.д.). | **(Исправлено)**. Конфигурация в `profile-service` и `vacancy-service` была унифицирована с использованием `pydantic-settings`. Теперь оба сервиса корректно загружают все необходимые переменные из `.env` файла. |
| **Бэкенд (auth-service)** | `app/utils/jwt.py` | **Жестко закодированный `SECRET_KEY`** для создания JWT. | **(Исправлено)**. Ключ вынесен в переменные окружения. **Необходимо**: сгенерировать сложный, случайный ключ для продакшена и убедиться, что он используется всеми сервисами. |
| **Фронтенд** | `src/api/auth.js` (и другие) | **Жестко закодированный URL API** (`http://127.0.0.1:8000`). | Вынести URL в переменные окружения (`.env` файл). Использовать `import.meta.env.VITE_API_URL`. |
| **Фронтенд** | `src/context/AuthContext.jsx` | **Хранение JWT в `localStorage`**. | Для повышения защиты от XSS-атак, хранить JWT в `httpOnly` cookie. Это требует изменений на бэкенде (установка cookie при логине). |

## 2. Рекомендации по улучшению (Best Practices)

Эти пункты не являются критическими уязвимостями, но их исправление значительно улучшит качество, надежность и поддерживаемость кода.

| Сервис/Часть | Файл | Проблема | Рекомендация по исправлению |
| :--- | :--- | :--- | :--- |
| **Бэкенд (Все сервисы с БД)** | `app/main.py` | **Автоматическое создание таблиц** (`Base.metadata.create_all`). | Внедрить систему миграций **Alembic** для управления изменениями схемы БД в продакшене. |
| **Бэкен-д (api-gateway)** | `app/routes/auth.py` | **Проксирование всех HTTP-заголовков** от клиента. | Создать "белый список" заголовков, которые можно безопасно проксировать, вместо того чтобы копировать все подряд. |
| **Бэкенд (profile-service)** | `app/main.py` | **Раздача статических файлов** через FastAPI. | В продакшене настроить веб-сервер (Nginx) для раздачи статики или использовать объектное хранилище (S3, MinIO). |
| **Бэкенд (profile-service)** | `app/core/messaging.py` | **Отсутствие обработки "мертвых" сообщений** в RabbitMQ. | Реализовать паттерн **Dead Letter Queue (DLQ)** для сообщений, которые не удалось обработать, чтобы избежать их потери. |
| **Фронтенд** | `src/context/AuthContext.jsx` | **Отсутствие обработки ошибок** при логине. | Обернуть вызовы API в `try...catch` и предоставлять пользователю обратную связь в случае ошибки. |
| **Фронтенд** | `src/api/*.js` | **Неполная обработка HTTP-ошибок** (отсутствие `res.ok`). | Добавить проверку `if (!res.ok)` во все `fetch` запросы и выбрасывать исключение для централизованной обработки. |
| **Фронтенд** | `src/api/*.js` | **Повторяющийся код для `fetch`**. | Создать централизованный HTTP-клиент (например, с помощью `axios` или обертки над `fetch`) для настройки базового URL и автоматического добавления заголовков. |

## 3. Косметические замечания и мелкие улучшения

| Сервис/Часть | Файл | Проблема | Рекомендация по исправлению |
| :--- | :--- | :--- | :--- |
| **Бэкенд (authservice)** | `app/db/session.py` | **Включенное логирование SQL-запросов** (`echo=True`). | Выключать `echo=True` в продакшен-режиме, чтобы избежать излишнего логирования и снижения производительности. |
| **Бэкенд (authservice)** | `app/auth/routes.py` | **Использование статуса 400** для существующего пользователя. | Для конфликта ресурсов (пользователь уже существует) более семантически верным будет статус `409 Conflict`. |
